After

openapi.yaml 
openapi: 3.1.0
paths:
  /users:
    post:
      requestBody:
        content:
          application/json:
            schema:
              type: object
              properties:
                email:
                  $ref: 'https://schemas.sourcemeta.com/sourcemeta/std/1.0.0/ietf/email/address'
                phone:
                  $ref: 'https://schemas.sourcemeta.com/sourcemeta/std/1.0.0/ietf/rfc3966/tel-uri'
                country:
                  $ref: 'https://schemas.sourcemeta.com/sourcemeta/std/1.0.0/iso/3166-1/alpha2'
                currency:
                  $ref: 'https://schemas.sourcemeta.com/sourcemeta/std/1.0.0/iso/4217/code'
                website:
                  $ref: 'https://schemas.sourcemeta.com/sourcemeta/std/1.0.0/ietf/rfc3986/uri'
                created_at:
                  $ref: 'https://schemas.sourcemeta.com/sourcemeta/std/1.0.0/iso/8601/datetime'
                language:
                  $ref: 'https://schemas.sourcemeta.com/sourcemeta/std/1.0.0/ietf/rfc5646/language-tag'

Frequently Asked Questions

Reference the schemas using the $ref keyword with HTTPS URLs from our hosting service:

schema:
  type: object
  properties:
    email:
      $ref: 'https://schemas.sourcemeta.com/sourcemeta/std/1.0.0/ietf/email/address'

Alternatively, download the schemas from GitHub Releases and reference them locally using relative paths, or add them as a git submodule to your repository.

Absolutely! These schemas are designed to be building blocks. You can reference them from your existing schemas using $ref, compose them with allOf, anyOf, or oneOf, or extend them with additional constraints.

Start by replacing common patterns (email, dates, URIs) and gradually adopt more schemas as you refactor your API specifications.

All schemas target JSON Schema 2020-12, which is the dialect used by OpenAPI v3.1 and later. Support for earlier JSON Schema dialects will be added in future releases.

We regularly update schemas to reflect new standards, fix bugs, and add coverage for additional specifications. Commercial license holders receive all updates for free. We follow semantic versioning, so you can safely upgrade to patch and minor versions without breaking changes.
